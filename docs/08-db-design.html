1 ER図

ER図 必須
テーブル間のリレーションと役割を文章で整理します。

admins：管理画面にログインする管理者アカウントを管理する。

admins（管理者）

└ 管理画面を利用する管理者情報を管理する

reservations（予約情報）

└ 予約者名、予約日、時間、人数、ステータスなどを管理する

inquiries（お問い合わせ）

└ 氏名、受信日、件名、内容、ステータス、返信内容を管理する

2 テーブル定義

テーブル定義：reservations 必須
予約フォームから送信される予約データを管理するテーブル。
カラム名 データ型 NULL デフォルト 制約 説明
id BIGINT NO AUTO_INCREMENT PK 主キー
customer_name VARCHAR(100) NO NULL NOT NULL 予約者名
customer_email VARCHAR(255) NO NULL NOT NULL メールアドレス
customer_phone VARCHAR(20) YES NULL - 電話番号
reservation_date DATE NO NULL NOT NULL 予約日
reservation_time VARCHAR(20) NO NULL NOT NULL 予約時間
people_count INT NO 1 NOT NULL 人数
status VARCHAR(20) NO '予約受付' NOT NULL 予約ステータス
note TEXT YES NULL - 備考
created_at DATETIME NO CURRENT_TIMESTAMP NOT NULL 作成日時
updated_at DATETIME NO CURRENT_TIMESTAMP NOT NULL 更新日時
テーブル定義：inquiries 必須
問い合わせフォームから送信される問い合わせ内容と対応履歴を管理するテーブル。
カラム名 データ型 NULL デフォルト 制約 説明
id BIGINT NO AUTO_INCREMENT PK 主キー
name VARCHAR(100) NO NULL NOT NULL 氏名
email VARCHAR(255) NO NULL NOT NULL メールアドレス
phone VARCHAR(20) YES NULL - 電話番号
received_date DATE NO NULL NOT NULL 受付日
subject VARCHAR(200) NO NULL NOT NULL 件名
message TEXT NO NULL NOT NULL 問い合わせ内容
status VARCHAR(20) NO '未対応' NOT NULL 対応状況
reply_subject VARCHAR(200) YES NULL - 返信件名
reply_message TEXT YES NULL - 返信内容
created_at DATETIME NO CURRENT_TIMESTAMP NOT NULL 作成日時
updated_at DATETIME NO CURRENT_TIMESTAMP NOT NULL 更新日時

3 インデックス設計

インデックス一覧 必須
テーブル名 インデックス名 対象カラム 種類 設定理由
reservations idx_reservations_date reservation_date INDEX 予約日で検索するため
reservations idx_reservations_status status INDEX ステータスで絞り込むため
reservations idx_reservations_email customer_email INDEX メールアドレスで検索するため
inquiries idx_inquiries_date received_date INDEX 受付日で検索するため
inquiries idx_inquiries_status status INDEX 対応状況で絞り込むため
admins idx_admins_login_id login_id UNIQUE ログインIDの重複を防ぐため
