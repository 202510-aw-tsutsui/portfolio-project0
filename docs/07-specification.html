1 システム概要

システムの目的 必須
このシステムが解決する課題と提供する価値

本システムは、香りのワークショップ「inori」の情報を分かりやすく伝え、予約までスムーズに進めるためのWebサイトです。

利用者は、ワークショップ内容、料金、日程、アクセス、よくある質問などを確認しながら予約できます。

また、お問い合わせフォームや予約確認ページも用意し、利用者が安心してサービスを利用できるようにしています。

予約ページでは、入力・決済・確認・完了の流れで予約を進められるようにしています。

4 機能詳細仕様

機能仕様テンプレート（コピーして使用）

機能名：予約入力機能必須
機能ID：F-003

利用者が予約フォームに必要な内容を入力する機能です。

お名前、メールアドレス、日程、時間、参加人数などを入力し、次の画面へ進みます。

入力後は決済方法の選択画面に進み、その後に確認画面と完了画面を表示します。

必須項目が未入力の場合は、正しく予約を進められないようにします。

入力項目一覧 必須
項目名 データ型 必須 バリデーション 備考
お名前（フリガナ） String — 任意入力 カタカナで入力
お名前 String ○ 必須チェック 利用者氏名を入力
メールアドレス String ○ 必須チェック ローカル部、ドメイン選択、その他入力あり
電話番号 String — 任意入力 電話番号を入力
予約日 Date ○ 必須チェック 日付を選択
予約時間 String — 必須チェック 選択式
参加人数 nteger ○ 必須チェック 1名〜6名から選択
備考欄 String — 任意入力 自由記入
キャンセルポリシー同意 Boolean ○ チェック必須 同意しないと進めない

5 非機能要件

セキュリティ要件 必須
認証・認可・データ保護に関する要件

予約フォームとお問い合わせフォームでは、必要な項目を正しく入力しないと送信できないようにします。

メールアドレスや氏名などの入力内容は、確認画面で見直せるようにします。

フォームには不要な自動入力を避ける設定を入れています。

利用者の個人情報は、予約やお問い合わせ対応のために使用します。

対応環境 必須
対応ブラウザ・デバイス

・対応ブラウザ：Google Chrome、Microsoft Edge

・対応デバイス：PC、スマートフォン

・レスポンシブ対応：スマートフォン表示に対応
