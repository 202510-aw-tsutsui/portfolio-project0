1 パッケージ構成

ディレクトリ構造 必須
src/main/java以下のパッケージ構造
src/main/java/
└── com.example.inori/
    ├── InoriApplication.java
    ├── controller/
    │   ├── ReservationController.java     # 予約管理
    │   └── InquiryController.java         # お問い合わせ管理
    ├── service/
    │   ├── ReservationService.java        # 予約業務ロジック
    │   └── InquiryService.java            # お問い合わせ業務ロジック
    ├── repository/
    │   ├── ReservationRepository.java     # 予約DBアクセス
    │   └── InquiryRepository.java         # お問い合わせDBアクセス
    ├── entity/
    │   ├── Reservation.java               # 予約エンティティ
    │   └── Inquiry.java                   # お問い合わせエンティティ
    └── form/
        ├── ReservationForm.java           # 予約入力フォーム
        └── InquiryForm.java               # お問い合わせ入力フォーム
パッケージの役割 必須
各パッケージの責務を説明
パッケージ 役割 使用するアノテーション
controller 予約管理画面・お問い合わせ管理画面のリクエスト受付と画面制御を行う @Controller, @GetMapping, @PostMapping, @RequestMapping
service 予約情報やお問い合わせ情報の登録・更新・削除などの業務ロジックを実装する @Service, @Transactional
repository 予約情報・お問い合わせ情報のデータベースアクセスを担当する @Repository
entity 予約情報・お問い合わせ情報のデータモデルを定義する @Data, @NoArgsConstructor, @AllArgsConstructor
form 画面から送信された入力値を受け取り、バリデーションを行う @Data, @NotBlank, @Email

3 Repository（リポジトリ）

ReservationRepository.java 必須
予約管理のインターフェース定義
┌────────────────────────────────────────────────┐
│              《Interface》                      │
│           ReservationRepository                │
│              @Repository                       │
├────────────────────────────────────────────────┤
│ + findAll(): List<Reservation>               │
│ + findById(id: Long): Reservation              │
│ + save(reservation: Reservation): void         │
│ + update(reservation: Reservation): void       │
│ + delete(id: Long): void                       │
│ + search(keyword: String): List<Reservation> │
└────────────────────────────────────────────────┘
メソッド 引数 戻り値 SQL種別 説明
findAll なし List<Reservation> SELECT 予約一覧を全件取得
findById id: Long Reservation SELECT ID指定で予約情報を取得
save reservation: Reservation void INSERT 予約情報を新規登録
update reservation: Reservation void UPDATE 予約情報を更新
delete id: Long void DELETE 予約情報を削除
search keyword: String List<Reservation> SELECT 氏名やメールアドレスで検索
InquiryRepository.java 必須
お問い合わせ管理のインターフェース定義
┌────────────────────────────────────────────────┐
│              《Interface》                      │
│             InquiryRepository                  │
│              @Repository                       │
├────────────────────────────────────────────────┤
│ + findAll(): List<Inquiry>                   │
│ + findById(id: Long): Inquiry                  │
│ + save(inquiry: Inquiry): void                 │
│ + update(inquiry: Inquiry): void               │
│ + delete(id: Long): void                       │
│ + search(keyword: String): List<Inquiry>     │
└────────────────────────────────────────────────┘
メソッド 引数 戻り値 SQL種別 説明
findAll なし List<Inquiry> SELECT お問い合わせ一覧を全件取得
findById id: Long Inquiry SELECT ID指定でお問い合わせ情報を取得
save inquiry: Inquiry void INSERT お問い合わせを新規登録
update inquiry: Inquiry void UPDATE お問い合わせ情報を更新
delete id: Long void DELETE お問い合わせ情報を削除
search keyword: String List<Inquiry> SELECT 氏名、件名、本文で検索

本システムでは、予約管理およびお問い合わせ管理を想定したクラス構成を設計した。 なお、実際の実装ではバックエンドおよびデータベースは使用しておらず、 JavaScriptとlocalStorageを用いたフロントエンド中心の構成としている。 そのため、本資料は将来的なSpring Boot実装を想定した設計内容である。
